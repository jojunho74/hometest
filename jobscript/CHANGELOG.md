# jobscript/index.html 작업 이력

> 공공기관 채용 경쟁률 분석 시각화 페이지 (`jobscript/index.html`)
> Supabase `alio_jobs` 테이블 데이터를 기반으로 직종별 경쟁률을 차트·테이블로 제공

---

## 연관 파일

| 파일 | 역할 |
|------|------|
| `jobscript/index.html` | 직종별 경쟁률 분석 메인 페이지 |
| `competition.html` | 공고 상세 경쟁률 페이지 (링크 연동) |
| `static/alio-admin.html` | 채용AI DATA 관리자 (데이터 파싱·저장) |

## DB 테이블

- **`alio_jobs`**: 직종별 채용 레코드 (dataset_name, job, region, grade, s1~s5 pass/total, ratio, stage_count)
- **`alio_source_cache`**: 원본 HTML 소스 캐시

## 주요 기능 정리

| 기능 | 설명 |
|------|------|
| 직종 태그 필터 | 우측 태그 클릭 → 해당 직종만 차트·테이블에 표시 |
| 직종 검색 | 검색창 입력 → 자동완성 드롭다운 (최대 20개) |
| 전체선택 바 | 검색 결과 직종 전체를 한번에 선택·해제 |
| 활성 필터 표시 | 현재 필터 상태를 초록 바로 항상 표시 |
| 차트 막대 클릭 | 해당 직종으로 필터 적용 |
| 공고명 클릭 | `ibtcc.kr/competition.html?ds=공고명` 새 탭으로 이동 |
| 정렬 | 경쟁률 높은순·낮은순·공고명순·응시인원순·직종명순 |
| 경쟁률 칩 | ≥80:1 빨강(hot) / ≥30:1 노랑(mid) / 그 외 초록(low) |
| 3차 컬럼 | `stage_count >= 4` 인 경우에만 표시 (기본값 3) |

## stage_count 컬럼 (alio_jobs)

- `ALTER TABLE alio_jobs ADD COLUMN IF NOT EXISTS stage_count INTEGER DEFAULT 3;`
- 파서가 HTML에서 `[1-5]차|최종` 행 수를 직접 카운트하여 저장
- 기존 레코드(479→905개): 기본값 3 (재파싱 시 정확한 값으로 갱신)
- 4단계 공고(예: 한국서부발전): 갱신 버튼으로 재파싱 필요
